ChampionCreator は、複数の仮想敵シナリオ条件を同時に満たす能力ポイント配分を自動で計算するwebアプリです。

耐久調整、火力ライン、素早さラインを同じ作業画面で確認し、条件を満たす候補の使用SP、残りSP、最も厳しい条件を比較できます。

公開URL: <https://suisui-swimmy.github.io/ChampionCreator/>

詳しい操作説明は Wiki を参照してください。

Wiki: <https://github.com/suisui-swimmy/ChampionCreator/wiki>

## できること

- 複数の仮想敵シナリオを同時に満たす `H / B / D` 耐久調整を探索
- 1つのシナリオ内で複数攻撃を順番に受ける条件を評価
- 必要耐久回数と必要生存率を指定
- 調整対象が仮想敵を倒すための `A / C` 火力ラインを探索
- 任意の相手や実数値を抜くための `S` ラインを探索
- 候補ごとの SP 配分、使用SP、残りSP、最厳条件、ダメージ詳細を表示
- 候補を調整対象へ適用
- 入力条件をブラウザ内のボックスへ保存・読み込み
- ホーム画面追加向けの PWA manifest に対応

## 使い方

1. 公開URLを開く
2. 調整対象のポケモン、性格、特性、持ち物、現在SPなどを入力する
3. 仮想敵シナリオに相手、技、攻撃側条件、必要耐久回数、生存率などを入力する
4. 必要ならシナリオを `耐久調整` / `火力調整` / `素早さ調整` に切り替える
5. `計算開始` を押す
6. 候補一覧から配分、残りSP、最厳条件、詳細ダメージを確認する
7. 使いたい候補の `適用` で調整対象へ反映する

詳しい入力項目、ボックス機能、火力ライン、素早さラインの説明は [Wiki](https://github.com/suisui-swimmy/ChampionCreator/wiki) を参照してください。

## 計算方針

- ダメージ計算は `@smogon/calc` を使用します
- 日本語名や別名は、計算前に Showdown canonical name へ解決します
- アプリ側は複数シナリオ、複数被弾、SP探索、生存率集計を管理します
- 公式画像やアイコンは表示用で、計算結果には影響しません

## SPについて

Pokemon Champions の Stat Points / SP を探索単位にしています。

- 各能力は `0..32 SP`
- 6能力合計は `66 SP` まで
- `H / B / D` は同時探索
- `A / C / S` は入力済みSP、火力ライン、素早さラインを固定条件として扱います
- `@smogon/calc` へ渡す直前に、実数値が一致する Showdown EV 相当へ変換します

## ブラウザ内保存

ボックス機能で、現在の調整対象、シナリオ、火力ライン、素早さラインなどの入力条件をブラウザ内に保存できます。

保存データは同じブラウザ内の保存領域を使います。別端末、別ブラウザ、ブラウザデータ削除後には引き継がれません。

## 注意点

- 本ツールは非公式のファンツールです。ゲーム内の結果と完全に一致することを保証しません
- Pokemon Champions と Showdown の仕様差、未対応データ、特殊処理により、実際のゲーム内結果と異なる可能性があります
- 重要な調整は実機でも確認してください
- 一部のフォーム違い、技、特性、持ち物は未対応または仮対応の場合があります
- 条件が複雑なほど探索に時間がかかります

## 開発

Node.js 22 系を想定しています。

```powershell
npm install
npm run dev
```

主な確認コマンド:

```powershell
npm test
npm run build
npm run check
```

`npm run check` は、日本語データ、ポケモン別特性、画像、テスト、production build をまとめて確認します。
