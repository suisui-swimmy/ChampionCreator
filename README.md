# ChampionCreator

ChampionCreator は、複数の仮想敵から受けるダメージをまとめて確認し、すべての条件を満たす `H / B / D` の SP 配分を探す自動耐久調整ツールです。

「この攻撃を確定で耐えたい」「物理技と特殊技の両方を見ながら、できるだけ少ない SP で調整したい」といった場面で使えます。

## できること

- 複数の仮想敵シナリオを同時に判定
- 1つのシナリオ内で、複数回・複数種類の攻撃を続けて受ける条件を設定
- 必要な耐久回数と生存率を指定
- 条件を満たす合法な `H / B / D` 配分を自動探索
- 調整対象が仮想敵を単発で倒すための火力ラインを算出
- 任意のすばやさラインを抜くための `S` ラインを算出
- 候補ごとの使用 SP、残り SP、最も厳しい条件、被ダメージを表示
- 選んだ候補を調整対象へ反映
- 入力条件を JSON で保存・共有・復元

ダメージ計算には `@smogon/calc` の第9世代計算を使用しています。

## 基本的な使い方

### 1. 調整対象を入力する

画面左側の「調整対象」に、耐久調整したいポケモンの情報を入力します。

- ポケモン
- 性格
- 特性
- 持ち物
- テラタイプとテラスタルの発動状態
- 個体値
- 現在の SP 配分

`H / B / D` は自動探索の対象です。`A / C / S` に入力した SP は固定値として扱われ、合計 SP の予算に含まれます。`火力調整` シナリオがある場合は必要な `A / C` ライン、`素早さ調整` シナリオがある場合は必要な `S` ラインも、候補一覧の計算時だけ固定SPへ合成されます。

### 2. 仮想敵シナリオを入力する

画面右側の「仮想敵シナリオ」に条件を入力します。各シナリオは、行右上の `耐久調整` / `火力調整` / `素早さ調整` で用途を切り替えます。

`耐久調整` では、受けたい攻撃の条件を入力します。

- 攻撃するポケモン
- 技
- 性格、特性、持ち物
- 攻撃側の SP
- 攻撃回数
- 必要耐久回数
- 必要生存率

天候、フィールド、能力ランク、壁、急所、てだすけ、状態異常などは「詳細補正」から設定できます。

`火力調整` では、調整対象が単発で倒したい仮想敵と技、必要な KO 率、仮想敵側の `H / A / B / C / D / S` SP を入力します。

`素早さ調整` では、抜きたい仮想敵または直接指定した任意S値、ランク、道具倍率、特性倍率、おいかぜなどを入力します。

### 3. 必要ならシナリオや攻撃を追加する

- 別の相手や別条件も同時に満たしたい場合は、シナリオを追加します
- 同じ場面で攻撃を続けて受ける場合は、同じシナリオ内に攻撃カードを追加します
- 一時的に計算から外したいシナリオは、削除せず無効にできます

同じシナリオ内の攻撃は、上から順番に受ける累積ダメージとして判定されます。

### 4. 「計算開始」を押す

入力内容に問題がなければ探索が始まり、進捗バーが表示されます。条件が多い場合や生存率計算が重い場合は、完了まで時間がかかることがあります。

探索中は「キャンセル」で中断できます。

### 5. 候補を確認する

候補一覧には、条件を満たした配分だけが表示されます。

| 表示 | 意味 |
| --- | --- |
| `H / A / B / C / D / S` | 候補適用後の SP 配分。`H / B / D` は候補値、`A / C` は固定値または火力ライン、`S` は固定値 |
| 使用SP | 候補適用後の6能力合計SP |
| 残りSP | 合計66 SPから候補適用後のSPを引いた残り |
| 最厳条件 | 候補にとって最も余裕が少ないシナリオ |
| 適用 | 候補の `H / A / B / C / D / S` を調整対象へ反映 |

候補行を開くと、シナリオごとの生存率、PASS判定、被ダメージの詳細を確認できます。

候補が0件の場合は、必要耐久回数、生存率、固定済みの `A / C / S` などを見直してください。

### 6. 火力ラインを確認する

各シナリオを `火力調整` に切り替えると、候補一覧の上に「火力ライン結果」が表示されます。調整対象が任意の仮想敵を単発で倒すための SP ラインを確認できます。

- 仮想敵
- 技
- 必要な KO 率
- 仮想敵側の SP、状態、ランク、壁、天候、フィールド

をシナリオ内に入力すると、`Aライン` / `Cライン` / `Bライン` / `Hライン` / `相手A参照` のように、技が参照する能力ごとの結果が表示されます。

`Aライン` と `Cライン` は候補一覧の順位へ自動で統合されます。`ボディプレス` の `Bライン` や `いのちがけ` の `Hライン` は、候補の `H / B / D` 最低条件として扱われます。`相手A参照` や固定ダメージなど、調整対象のSPで改善できない条件は現在条件での達成/未達として表示されます。

### 7. Sラインを確認する

各シナリオを `素早さ調整` に切り替えると、候補一覧の上に「Sライン結果」が表示されます。任意のすばやさラインに対して、必要な `S` SP と最終Sを確認できます。

- `確定抜き` では、仮想敵の性格、ランク、持ち物、特性、天候、フィールド、おいかぜなどから相手の最終Sを計算し、`+1` など任意の加算値を指定できます
- `任意S値` では、データ未対応や特殊条件でも入力した数値を相手最終Sとして扱えます
- 道具倍率と特性倍率の `なし` は自動判定、`2倍` / `1.5倍` / `0.5倍` は手動上書きです

結果の「適用」を押すと、必要な `S` SP が調整対象へ反映されます。候補一覧の計算では、入力済みの `S` と `Sライン結果` の大きい方が固定SPとして扱われます。

## SP入力について

Pokemon Champions の SP を基準にしています。

- 各能力は `0..32 SP`
- 6能力の合計は `66 SP` まで
- `H / B / D` を同時に探索
- `A / C` は入力値と火力ラインの大きい方、`S` は入力値とSラインの大きい方で固定

入力欄へ `252` などの Showdown EV を入れた場合は、対応する SP へ変換されます。

## 条件を保存・共有する

ヘッダーの「コピー」を押すと、現在の調整対象とシナリオ条件を JSON としてクリップボードへコピーできます。`火力調整` や `素早さ調整` に切り替えたシナリオの条件も同じ JSON に含まれます。

復元するときは次の手順です。

1. 「条件JSON」を押す
2. 保存しておいた JSON を入力欄へ貼り付ける
3. 「読込」を押す

読み込むと現在の入力内容が置き換わります。大事な条件は、先に「コピー」で保存しておくのがおすすめです。

## 計算結果について

- 最終判定はすべて `@smogon/calc` のダメージ結果を使います
- 日本語名は内部で Showdown の計算用名称へ変換してから計算します
- 公式画像は表示専用で、計算結果には影響しません
- 候補は火力ラインと耐久条件を同時に満たしたうえで、使用 SP が少ないものを優先して表示します
- 現在は上位20件までを表示します

Pokemon Champions と Showdown で仕様差がある場合、実際のゲーム内結果と一致しない可能性があります。重要な調整は実機でも確認してください。

## 対応していないこと・注意点

- URLだけで条件を共有する機能はありません。条件JSONを使用してください
- 一部のフォーム違いや同名候補は、入力を確定できず警告になる場合があります
- 条件が複雑なほど探索時間が長くなります
- ブラウザや端末をまたぐ自動保存には対応していません
- アクセス状況の把握と改善のため、Google Analyticsを使用しています

## 開発・動作確認

Node.js 22 を使用します。

```powershell
npm install
npm run dev
```

変更後の一括確認:

```powershell
npm run check
```

`npm run check` は、日本語データ、ポケモン別特性、画像、テスト、production buildをまとめて確認します。
